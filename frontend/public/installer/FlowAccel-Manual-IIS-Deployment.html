Before you start
🛠️
Run every command as Administrator
You'll be enabling Windows roles, writing to E:\, modifying IIS bindings, and loading SQL into PostgreSQL. Open PowerShell / Command Prompt with Run as administrator for every step.
🌐
Port swap — IIS Default Web Site → 8081, FlowAccel takes port 80
Move the IIS Default Web Site off port 80 onto port 8081 (so it gets out of the way), then bind the FlowAccel site to the freed-up port 80. End users open http://<server-ip>/ with no port suffix.
💾
All project files live on the E: drive
Deployment root is E:\FlowAccel\. The IIS application pool user must have NTFS permissions on this folder — granted manually in step 6.
🗄️
Database: PostgreSQL, database name "flowaccel"
Load the schema dump with psql.exe -U postgres -d flowaccel -f <your-sql-file> from the PostgreSQL bin folder.
Step-by-step deployment
1
Enable IIS (Internet Information Services)

Open Server Manager → Manage → Add Roles and Features, or run the PowerShell command below as administrator.

Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServerRole, IIS-WebServer, IIS-CommonHttpFeatures, IIS-StaticContent, IIS-DefaultDocument, IIS-DirectoryBrowsing, IIS-HttpErrors, IIS-HttpRedirect, IIS-ApplicationDevelopment, IIS-NetFxExtensibility45, IIS-HealthAndDiagnostics, IIS-HttpLogging, IIS-Security, IIS-RequestFiltering, IIS-Performance, IIS-WebServerManagementTools, IIS-ManagementConsole, IIS-IIS6ManagementCompatibility, IIS-Metabase, IIS-WindowsAuthentication -All

Wait for the install to finish, then verify by opening a browser to http://localhost — you should see the default IIS landing page.

💡 On Windows 10 / 11 desktop editions, use Control Panel → Programs → Turn Windows features on or off instead of Server Manager. Check every node under Internet Information Services.
2
Move the IIS Default Web Site from port 80 → 8081 (frees up port 80 for FlowAccel)

The IIS Default Web Site ships bound to port 80. We move it onto 8081 so port 80 is free, then bind the real FlowAccel site to port 80 in step 5.

Open IIS Manager (inetmgr in Run).

  1. In the left tree, expand the server node → Sites → click Default Web Site.
  2. On the right panel, click Bindings….
  3. Select the existing http binding (Port 80) → click Edit….
  4. Change Port from 80 to 8081 → click OK → Close.
  5. Click Restart on the right-side Manage Website panel.

Or do it from the command line:

%windir%\system32\inetsrv\appcmd set site "Default Web Site" /bindings:"http/*:8081:" iisreset

Verify in a browser at http://localhost:8081 (the placeholder IIS page should now answer on the new port). Port 80 is now free — FlowAccel will claim it in step 5.

⚠️ Open both ports in Windows Firewall — port 80 (FlowAccel) and 8081 (Default Web Site):
netsh advfirewall firewall add rule name="FlowAccel HTTP 80" dir=in action=allow protocol=TCP localport=80 netsh advfirewall firewall add rule name="IIS Default 8081" dir=in action=allow protocol=TCP localport=8081
3
Enable IIS roles and security permissions

In IIS Manager, with the server node selected, double-click Authentication and confirm:

  • Anonymous Authentication — Enabled
  • Windows Authentication — Enabled (if your environment uses AD SSO)

Then double-click Authorization Rules and ensure Allow All Users is present.

Make sure these role services are installed (run as administrator):

Enable-WindowsOptionalFeature -Online -FeatureName IIS-StaticContent, IIS-RequestFiltering, IIS-WindowsAuthentication, IIS-ManagementConsole -All

Restart IIS to apply changes:

iisreset
4
Install all third-party dependencies from the zip

The FlowAccel package zip ships a packages\ folder (sometimes named third-party\ or installers\) with all required runtimes.

Open the packages folder and install each installer one by one — right-click each file → Run as administrator. Install Node.js FIRST, then the rest in the order shown below:

  1. Node.js 18.x — node-v18.x.x-x64.msi  ← install this one FIRST
  2. URL Rewrite Module for IIS — rewrite_amd64_en-US.msi
  3. iisnode — iisnode-full-v0.2.x-x64.msi
  4. PostgreSQL 15 — postgresql-15.x-windows-x64.exe
    During setup, set the postgres superuser password — remember it, you'll need it in step 8.
  5. Visual C++ Redistributable (if included in the folder) — vc_redist.x64.exe
  6. Application Request Routing (ARR) for IIS — requestRouter_amd64.msi
    Enables IIS to reverse-proxy requests to the Node backend.
  7. PM2 (Node process manager) — installed via npm after Node.js is set up:
    npm install -g pm2

Don't skip any file in the packages\ folder — go through them one by one until every installer has been run.

💡 After installing each MSI/EXE, you don't need to reboot — but do close and reopen any Command Prompt / PowerShell window so the new PATH entries become visible.
5
Copy files to E:\ and create the FlowAccel site on port 80

Create the deployment root and copy the FlowAccel application files into it:

mkdir E:\FlowAccel xcopy /E /I /Y "<path-to-extracted-zip>\app" "E:\FlowAccel\"

After the copy, you should have this layout:

E:\FlowAccel\ ├─ dist\ ← static frontend (served by IIS) ├─ api\ ← Node.js backend (iisnode entry point) ├─ web.config ← IIS site config ├─ package.json └─ node_modules\ ← run "npm install --production" if missing

Now create a new IIS site for FlowAccel bound to port 80 (this is the URL end users will open — no port suffix needed):

  1. Open IIS Manager → right-click Sites → Add Website….
  2. Site name: FlowAccel
  3. Physical path: E:\FlowAccel\dist (or the path that contains web.config)
  4. Binding: Type = http, IP = All Unassigned, Port = 80, Host name = (blank)
  5. Click OK.

Or do it from the command line:

%windir%\system32\inetsrv\appcmd add site /name:"FlowAccel" /physicalPath:"E:\FlowAccel\dist" /bindings:"http/*:80:" iisreset
💡 After this step you should have two sites in IIS Manager: Default Web Site on port 8081 (placeholder) and FlowAccel on port 80 (the real app).
⚠️ If iisreset fails with "port 80 already in use", you didn't move the Default Web Site off port 80 in step 2 — go back and complete step 2 before retrying.
6
Grant IIS user permissions on E:\FlowAccel

IIS runs the site under the application pool identity — by default IIS AppPool\DefaultAppPool, plus the built-in groups IIS_IUSRS and IUSR. All three need NTFS access on the folder.

Via Explorer (matches the screenshot):

  1. Right-click E:\FlowAccel → Properties → Security tab → Edit….
  2. Click Add…. In Enter the object names to select, type each name below (one at a time), click Check Names, then OK:
    • IIS_IUSRS
    • IUSR
    • IIS AppPool\DefaultAppPool  (Locations… → choose the local machine first)
    • NETWORK SERVICE
  3. For each added principal, tick Modify, Read & execute, List folder contents, Read, and Write under Allow → OK.

Or, equivalently, from an elevated PowerShell:

icacls "E:\FlowAccel" /grant "IIS_IUSRS:(OI)(CI)M" /T icacls "E:\FlowAccel" /grant "IUSR:(OI)(CI)M" /T icacls "E:\FlowAccel" /grant "IIS AppPool\DefaultAppPool:(OI)(CI)M" /T icacls "E:\FlowAccel" /grant "NETWORK SERVICE:(OI)(CI)M" /T
⚠️ If you use a non-default app pool (e.g. FlowAccelPool), substitute IIS AppPool\FlowAccelPool in the commands above.
7
Verify all dependencies are installed

Open a new elevated Command Prompt (so PATH is refreshed) and run each check. All must succeed.

  • 🟢
    Node.js
    node --version → v18.x.x
  • 🟢
    npm
    npm --version → 9.x or 10.x
  • 🟢
    iisnode module loaded by IIS
    %windir%\system32\inetsrv\appcmd list module /name:iisnode → MODULE "iisnode" listed
  • 🟢
    URL Rewrite
    %windir%\system32\inetsrv\appcmd list module /name:RewriteModule → MODULE "RewriteModule" listed
  • 🟢
    PostgreSQL
    psql --version → psql (PostgreSQL) 15.x
  • 🟢
    IIS Default Web Site (port 8081)
    curl -I http://localhost:8081 → HTTP/1.1 200 OK
  • 🟢
    FlowAccel site (port 80)
    curl -I http://localhost → HTTP/1.1 200 OK
💡 If psql is not found, add C:\Program Files\PostgreSQL\15\bin to system PATH and reopen the terminal.
8
Dump the .sql file into PostgreSQL (via pgAdmin + PowerShell)

Once all third-party dependencies from step 4 are installed, load the SQL dump into the database. This is a three-part procedure: create the DB in pgAdmin → open PowerShell as administrator → run psql from the bin folder.

8a — Create the database in pgAdmin

  1. Open pgAdmin 4 (installed with PostgreSQL in step 4).
  2. In the left tree, expand Servers → PostgreSQL 15 and enter the postgres password when prompted.
  3. Right-click Databases → Create → Database….
  4. Enter the Database name (e.g. flowaccel) → click Save.

Remember the exact name you used — you'll pass it to -d in step 8c.

8b — Open PowerShell as Administrator

Press Start → type PowerShell → right-click Windows PowerShell → Run as administrator.

⚠️ A normal (non-admin) PowerShell window will fail with permission errors when reading the SQL file or writing logs. Always run as administrator for this step.

8c — Go to the PostgreSQL bin folder and load the SQL file

In the elevated PowerShell window, change into the PostgreSQL bin directory of the project (this is the folder where psql.exe lives):

cd "C:\Program Files\PostgreSQL\15\bin"

Then run psql.exe with:

  • -U postgres — the superuser name
  • -d <database> — the database name you created in pgAdmin in step 8a
  • -f "<path-to-your-.sql-file>" — the full path to the dump file
.\psql.exe -U postgres -d flowaccel -f "C:\Users\Administrator\Desktop\yourfile.sql"

Replace flowaccel with whatever name you used in pgAdmin, and C:\Users\Administrator\Desktop\yourfile.sql with the real path to your .sql file. When prompted, enter the postgres password.

💡 Verify the load succeeded — list the tables:
.\psql.exe -U postgres -d flowaccel -c "\dt"
🚨 If you see permission denied or role "postgres" does not exist, the PostgreSQL service isn't running. Start it from services.msc → postgresql-x64-15 → Start.

8d — After the SQL import: create the app DB user, grant privileges & enable the IIS proxy

Only run these once the .sql file has imported successfully. In the same elevated PowerShell, run the block below. It creates the jotflow application user (idempotent — safe to re-run), grants it access to the flowaccel database, and enables the IIS ARR reverse proxy.

& "C:\Program Files\PostgreSQL\15\bin\psql.exe" -U postgres -h 127.0.0.1 -c "DO `$`$ BEGIN IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname='jotflow') THEN CREATE USER jotflow WITH PASSWORD 'hope9886@'; END IF; END `$`$; GRANT ALL PRIVILEGES ON DATABASE flowaccel TO jotflow;" & "C:\Program Files\PostgreSQL\15\bin\psql.exe" -U postgres -h 127.0.0.1 -d flowaccel -c "GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO jotflow; GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO jotflow; ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO jotflow;"

Then enable the IIS Application Request Routing proxy (installed in step 4):

Set-WebConfigurationProperty -PSPath 'MACHINE/WEBROOT/APPHOST' -Filter 'system.webServer/proxy' -Name 'enabled' -Value 'True'
⚠️ If the combined block above errors out, run the three commands separately instead:
REM 1 — create the user & "C:\Program Files\PostgreSQL\15\bin\psql.exe" -U postgres -h 127.0.0.1 -c "CREATE USER jotflow WITH PASSWORD 'hope9886@';" REM 2 — grant privileges on the flowaccel database & "C:\Program Files\PostgreSQL\15\bin\psql.exe" -U postgres -h 127.0.0.1 -d flowaccel -c "GRANT ALL PRIVILEGES ON DATABASE flowaccel TO jotflow; GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO jotflow; GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO jotflow; ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO jotflow;" REM 3 — enable the IIS ARR proxy Set-WebConfigurationProperty -PSPath 'MACHINE/WEBROOT/APPHOST' -Filter 'system.webServer/proxy' -Name 'enabled' -Value 'True'
💡 The jotflow user / password here must match the DB credentials in the backend .env (see the notes at the bottom of this guide).
9
Restart the backend server

After the database is loaded and the .env values are in place (see the notes at the bottom), restart the Node backend so it picks up the fresh config. In the elevated PowerShell, change into the backend folder and start the server:

cd E:\FlowAccel\api node server.js
💡 For a long-running, auto-restarting process, use PM2 instead (installed in step 4):
pm2 restart all  (or pm2 start server.js --name flowaccel the first time).
⚠️ The backend won't authenticate or load workflow data until the .env keys (MICROSOFT_CLIENT_ID, MICROSOFT_CLIENT_SECRET, MICROSOFT_TENANT_ID, MICROSOFT_REDIRECT_URI, JOTFORM_API_KEY, JOTFORM_TEAM_ID) are filled in — see the notes below.
10
Final smoke test

Open a browser on the server and visit the FlowAccel site on port 80 (no port suffix needed):

http://localhost/

You should see the FlowAccel login page. From another machine on the LAN, use the server's IP:

http://<server-ip>/

If the page loads, all steps are complete and FlowAccel is live on IIS.

💡 Reminder — port 8081 shows the placeholder IIS Default Web Site, port 80 (the default) is the real FlowAccel app.
📝 Important — Backend .env configuration

All settings — the JotForm API key, Microsoft credentials, form/team IDs — live in the backend .env file inside the backend folder (E:\FlowAccel\api\.env). Anytime you need to change a value, open that .env file and edit it there — then restart the backend. Copy-paste the values you saved in the Step 0 form at the top of this guide straight into the matching keys below. FlowAccel will not authenticate or pull workflow data until every value is filled in:

MICROSOFT_CLIENT_ID=<Microsoft Client ID — from Step 0> MICROSOFT_CLIENT_SECRET=<Microsoft Client Secret — from Step 0> MICROSOFT_TENANT_ID=<Microsoft Tenant ID — from Step 0> MICROSOFT_REDIRECT_URI=<Microsoft Redirect URL — from Step 0> JOTFORM_API_KEY=<client's JotForm API key> JOTFORM_TEAM_ID=<JotForm team ID>
  • MICROSOFT_CLIENT_ID / MICROSOFT_CLIENT_SECRET / MICROSOFT_TENANT_ID / MICROSOFT_REDIRECT_URI — the client's Azure AD app registration values you entered and saved in Step 0 at the top of this guide. Use the copy icon there to paste each one here.
  • JOTFORM_API_KEY — JotForm → Account → API → create a key.
  • JOTFORM_TEAM_ID — the JotForm Enterprise team/workspace ID.
🚨 Keep the .env file private — never commit it or share the client secret / API key. After editing .env, restart the backend (pm2 restart all) so the new values load.
