             isError = Boolean

link: If it's any or all: Any or All


terms: JSON array defining the conditions to match (as below)


             id = id


             field = source element


             operator = (see operators list below)


             value = what to match (placeholder via unique name)


             isError = Boolean

             url = change thank you page url


             message = change thank you page text






disabled: 1 or NULL

12 Filtering 


Filter with OR on submissions

GET /API/sheets/#formid/sheet/#formid/view/#formid/rows



And then the filter parameter can be specified with an Array to filter using OR across that question ID, 

Eg:

filter=%7B%22q3:matches%22:\[%22foo%22,%22bar%22\]%7D



Would filter question 3 providing where the response was foo or bar



Filter Operators

When using the API you can use the filter key where available, there are operators for it that work with
strings, numbers, and dates these are:




Strings & Numbers:
matches

eq                                    Equals

ne                                    Not Equals

lt                                    Less Than

lte                                   Less Than and Equal

gt                                    Greater Than

gte                                   Greater Than and Equal


Dates
matches

lt                                    Less Than

lte                                   Less Than and Equal

gt                                    Greater Than

gte                                   Greater Than and Equal

sheets: 1
