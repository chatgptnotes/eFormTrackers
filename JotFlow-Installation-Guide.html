1. System Overview & Architecture

JotFlow is a workflow approval dashboard for the Government of Dubai Media Office. It connects to JotForm Enterprise (bettroi.jotform.com) and provides a modern approval interface with real-time updates.

ComponentRolePort
IIS (Internet Information Services)Serves the React frontend + reverse-proxies API/Socket calls to Node80
Node.js backend (via PM2)Express API, Socket.IO real-time, background poller3001
PostgreSQLDatabase: users, submissions, approvals, sessions5432
pgAdmin 4Database management GUI—

Traffic flow: Browser → IIS:80 → [static files] or [proxy → Node:3001 → PostgreSQL:5432]

ℹ

Offline-first: The ZIP contains all Windows installers. No internet is required on the target machine after extraction.

4. Step 1 — Enable IIS

⚠

Do this BEFORE installing Node.js or PostgreSQL. IIS must be active so that iisnode and URL Rewrite modules can register properly during installation.

A

Method A — PowerShell (Recommended)

Open PowerShell as Administrator (right-click Start → "Windows PowerShell (Admin)") and run:

# Enable IIS with all required sub-features for JotFlow
Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServer -All -NoRestart
Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebSockets -NoRestart
Enable-WindowsOptionalFeature -Online -FeatureName IIS-HttpCompressionDynamic -NoRestart
Enable-WindowsOptionalFeature -Online -FeatureName IIS-ManagementConsole -NoRestart

Wait for all commands to complete. Do NOT reboot yet — that is Step 4.

B

Method B — Control Panel (GUI)

  1. Press Win + R, type optionalfeatures, press Enter
  2. Scroll to Internet Information Services and tick the checkbox
  3. Expand it and ensure these sub-items are ticked:
    • Web Management Tools → IIS Management Console
    • World Wide Web Services → Application Development Features → WebSocket Protocol
    • World Wide Web Services → Common HTTP Features (all)
  4. Click OK and wait for Windows to apply changes
✔

Verify IIS is running: Open a browser and navigate to http://localhost/ — you should see the IIS default "Welcome" page. If you see it, IIS is active.

5. Step 2 — Install Node.js

1

Run the Node.js installer

  1. Navigate to 1_INSTALLERS\ inside the extracted ZIP
  2. Double-click node-v20.18.0-x64.msi
  3. Click Next on all screens — accept all defaults
  4. On the "Tools for Native Modules" screen: leave unchecked (not needed)
  5. Click Install, then Finish
ℹ

Do NOT verify Node.js in PowerShell yet — PATH updates only take effect after a reboot (Step 4).

6. Step 3 — Install PostgreSQL + pgAdmin

⚠

Critical: When asked for a superuser password during install, enter exactly: postgres
The setup script uses this password automatically. If you use a different password, you must edit the setup script before running it.

1

Run the PostgreSQL installer

  1. Navigate to 1_INSTALLERS\
  2. Double-click postgresql-16.6-windows-x64.exe
  3. Setup Wizard — click Next
  4. Installation Directory — leave as default (C:\Program Files\PostgreSQL\16), click Next
  5. Select Components — leave all checked (PostgreSQL Server, pgAdmin 4, Stack Builder, Command Line Tools), click Next
  6. Data Directory — leave as default, click Next
  7. Password — type postgres (remember this!), click Next
  8. Port — leave as 5432, click Next
  9. Locale — leave as default, click Next
  10. Click Next to start installation (takes 2–5 minutes)
  11. When asked to launch Stack Builder: uncheck and click Finish
✔

pgAdmin 4 is automatically installed alongside PostgreSQL — you don't need a separate installer.

7. Step 4 — Reboot the Machine

!

Reboot is MANDATORY

After installing Node.js and PostgreSQL, the system PATH must be refreshed. Without a reboot:

  • PowerShell will report node: command not found
  • PowerShell will report psql: command not found
  • The setup script (Step 6) will fail with prerequisite errors

Save all work and restart Windows now.

11. Post-Install Configuration

Configure JotForm API Key

The backend needs a JotForm API key to sync submissions. Edit C:\inetpub\flowaccel_backend\backend\.env (or wherever the backend was deployed — check 2_APP\backend\.env) and set:

JOTFORM_API_KEY=your-api-key-here
JOTFORM_TEAM_ID=260541093809054
JOTFORM_BASE=https://bettroi.jotform.com/API
JOTFORM_HOST=https://bettroi.jotform.com
ENABLE_POLLER=1
POLL_INTERVAL_MINUTES=5

After editing, restart the backend: pm2 restart jotflow-backend

Change Admin Password

  1. Login at http://localhost/ with the printed credentials
  2. Go to Settings → Profile
  3. Click Change Password
  4. Set a strong password and save

Connect pgAdmin to the Database

  1. Open pgAdmin 4 from the Start menu
  2. Right-click Servers → Register → Server
  3. General tab → Name: JotFlow Local
  4. Connection tab:
    • Host: localhost
    • Port: 5432
    • Database: jotflow
    • Username: postgres (superuser) or jotflow (app user)
    • Password: postgres
  5. Click Save

Set Windows Firewall Rule (for LAN access)

If other machines on the LAN need to access JotFlow at the server's IP:

# Allow port 80 inbound (run in Admin PowerShell)
New-NetFirewallRule -DisplayName "JotFlow HTTP" -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow

12. Error Reference & Fixes

ℹ

Every error listed here includes the exact message you will see and the exact fix. Work through them in order — most errors have a single root cause.

A. During Installation (setup-flowaccel.ps1)

⛔ "Not running as Administrator"

Message: [X] Not running as Administrator. Right-click PowerShell, Run as Administrator.

FIX Close PowerShell. Right-click the Start button → Windows PowerShell (Admin). Re-run the script.

⛔ "Node.js not found" / "node: command not found"

Cause: Node.js was installed but PATH has not refreshed yet, OR the installer didn't complete.

FIX 1 Reboot the machine. This is the most common fix.

FIX 2 If already rebooted, manually check: open PowerShell and run node -v. If still not found, re-run 1_INSTALLERS\node-v20.18.0-x64.msi and reboot again.

FIX 3 If node.exe exists but isn't in PATH: $env:Path += ";C:\Program Files\nodejs" in the same PowerShell session.

⛔ "psql not found in PATH"

Cause: PostgreSQL was installed but its bin folder isn't in PATH, OR a reboot is needed.

FIX 1 Reboot the machine.

FIX 2 Add PostgreSQL to PATH manually in the same PowerShell session:

$env:Path += ";C:\Program Files\PostgreSQL\16\bin"

FIX 3 Then re-run the setup script.

⛔ PostgreSQL connection refused / "could not connect to server"

Cause: PostgreSQL service is not running.

FIX Start the PostgreSQL service:

# In Admin PowerShell:
Start-Service -Name "postgresql*"
# OR use Services app: Win+R → services.msc → find "postgresql-x64-16" → Start
⛔ "password authentication failed for user postgres"

Cause: You used a different superuser password during PostgreSQL installation (not postgres).

FIX Set the correct password in the PowerShell session before running setup:

$env:PGPASSWORD = "your-actual-password"
powershell -ExecutionPolicy Bypass -File .\setup-flowaccel.ps1
⛔ "npm ci" fails / "npm ERR! Missing: package-lock.json"

Cause: The package-lock.json is missing from 2_APP\backend\.

FIX Run npm install instead:

cd C:\JotFlow-Setup\2_APP\backend
npm install --cache C:\JotFlow-Setup\4_NPM_CACHE --prefer-offline
⛔ "PM2 install failed" / PM2 not found after install

Cause: Global npm packages sometimes need a PATH refresh, or the pm2 tgz in the bundle had an issue.

FIX 1 Close and re-open PowerShell as Admin, then:

npm install -g pm2
pm2 -v

FIX 2 If no internet, install from bundled tgz:

npm install -g C:\JotFlow-Setup\1_INSTALLERS\npm\pm2-*.tgz
⛔ iisnode install failed / "iisnode registry key still missing"

Cause: iisnode MSI installer failed because IIS was not fully enabled when it ran.

FIX

  1. Enable IIS fully (Step 1 of this guide)
  2. Run the iisnode installer manually:
msiexec /i "C:\JotFlow-Setup\1_INSTALLERS\iisnode-full-v0.2.21-x64.msi" /qn
  • Re-run: powershell -ExecutionPolicy Bypass -File .\setup-flowaccel.ps1
    • ⛔ URL Rewrite / ARR install fails

    Cause: IIS is not enabled, or the MSI is blocked by Windows SmartScreen / AV.

    FIX Install manually:

    # Run both in Admin PowerShell:
msiexec /i "C:\JotFlow-Setup\1_INSTALLERS\rewrite_amd64_en-US.msi" /qn
msiexec /i "C:\JotFlow-Setup\1_INSTALLERS\requestRouter_amd64.msi" /qn

    Then re-run the setup script.

    B. After Installation — Browser Errors

    ⛔ http://localhost/ shows IIS default "Welcome" page

    Cause: The "Default Web Site" on port 80 is overriding the FlowAccel site.

    FIX

    # In Admin PowerShell:
& "$env:windir\system32\inetsrv\appcmd.exe" stop site "Default Web Site"
& "$env:windir\system32\inetsrv\appcmd.exe" start site "FlowAccel"
# OR open IIS Manager → Sites → right-click "Default Web Site" → Stop
    ⛔ http://localhost/ shows 502 Bad Gateway

    Cause: The Node.js backend on port 3001 is not running. IIS is up but the API backend is down.

    FIX 1 Check PM2 status and restart:

    pm2 status
pm2 restart jotflow-backend
pm2 logs jotflow-backend --lines 30

    FIX 2 If PM2 shows the process is erroring, check the backend log for the real error:

    type C:\JotFlow-Setup\2_APP\backend\logs\err.log

    FIX 3 If the .env file is missing, recreate it (see Post-Install Config section) and restart PM2.

    ⛔ http://localhost/ shows 403 Forbidden

    Cause: IIS app pool doesn't have read permission on the site folder.

    FIX

    # Grant IIS read permissions (Admin PowerShell):
icacls "C:\inetpub\flowaccel" /grant "IIS_IUSRS:(OI)(CI)RX" /T
icacls "C:\inetpub\flowaccel" /grant "IUSR:(OI)(CI)RX" /T
    ⛔ http://localhost/ shows 404 Not Found

    Cause: The IIS site is not pointing to the dist\ folder, or the frontend files were not deployed.

    FIX Re-run the IIS deployment script:

    cd C:\JotFlow-Setup
powershell -ExecutionPolicy Bypass -File .\deploy-to-iis.ps1
    ⛔ http://localhost/ shows 500 Internal Server Error

    Cause: URL Rewrite module is missing or ARR reverse-proxy is disabled.

    FIX 1 Install URL Rewrite + ARR (see install error above)

    FIX 2 Enable ARR proxy:

    & "$env:windir\system32\inetsrv\appcmd.exe" set config -section:system.webServer/proxy /enabled:'True' /commit:apphost
    ⛔ Login page loads but login fails with "Invalid credentials"

    Cause: Admin user was not seeded correctly, or you're using the wrong password.

    FIX Re-run the seed script to reset admin credentials:

    cd C:\JotFlow-Setup\2_APP\backend
$env:ADMIN_EMAIL = "admin@flowaccel.local"
$env:ADMIN_PASSWORD = "Admin@12345"
node db/seed-admin.js

    Then try logging in again with admin@flowaccel.local / Admin@12345

    ⛔ Login fails with "Database error" or "connection refused"

    Cause: PostgreSQL is not running, or the .env DATABASE_URL is wrong.

    FIX 1 Start PostgreSQL:

    Start-Service -Name "postgresql*"

    FIX 2 Check the .env file:

    type C:\JotFlow-Setup\2_APP\backend\.env

    The DATABASE_URL should be: postgresql://jotflow:jotflow@localhost:5432/jotflow

    C. Real-Time / Socket.IO Errors

    ⛔ Dashboard shows "Disconnected" or real-time updates don't work

    Cause: WebSocket connections are blocked — IIS WebSocket feature is disabled, or ARR isn't forwarding WebSocket upgrades.

    FIX 1 Enable IIS WebSockets feature:

    Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebSockets -NoRestart

    FIX 2 Verify web.config in C:\inetpub\flowaccel\ has Socket.IO rewrite rules — re-run deploy-to-iis.ps1 if missing.

    FIX 3 Test if backend is accessible directly:

    curl http://localhost:3001/api/health

    If this fails, restart PM2: pm2 restart jotflow-backend

    D. Data Sync / JotForm API Errors

    ⛔ No submissions appear in the dashboard

    Cause: JotForm API key is not set or is invalid.

    FIX

    1. Edit 2_APP\backend\.env — set the correct JOTFORM_API_KEY
    2. Restart backend: pm2 restart jotflow-backend
    3. Trigger a manual sync: go to Settings → Sync All in the dashboard
    4. Check poller logs: pm2 logs jotflow-backend --lines 50
    ⛔ "[poller] Poll failed: Jotform API error: 401"

    Cause: The JotForm API key is wrong or pointing to the wrong endpoint.

    FIX Verify the key works against the GDMO endpoint:

    # Test in PowerShell:
$headers = @{ "APIKEY" = "your-api-key-here" }
Invoke-RestMethod "https://bettroi.jotform.com/API/user" -Headers $headers

    If this returns responseCode: 200, the key is valid. Ensure JOTFORM_BASE=https://bettroi.jotform.com/API in .env.

    ⛔ "[poller] Poll failed: Jotform API error: 429"

    Cause: JotForm API rate limit hit. The poller is running too frequently.

    FIX Increase poll interval in .env:

    POLL_INTERVAL_MINUTES=15

    Then restart: pm2 restart jotflow-backend

    E. After Rebooting the Server

    ⛔ App stops working after Windows reboot

    Cause: PM2 is not set up to auto-start on boot, so the Node backend is dead after restart.

    FIX Set up PM2 auto-start (run once in Admin PowerShell):

    pm2 startup
# Run the command it prints, then:
pm2 save

    After this, PM2 will start automatically on every Windows boot.

    VERIFY After a reboot, run: pm2 status — should show online

    ⛔ PostgreSQL doesn't start after reboot

    Cause: PostgreSQL service is not set to Automatic start.

    FIX

    # Set PostgreSQL service to auto-start:
Set-Service -Name "postgresql-x64-16" -StartupType Automatic
Start-Service -Name "postgresql-x64-16"

    F. Permission Errors

    ⛔ "EACCES: permission denied" when writing uploads/logs

    Cause: The IIS app pool identity doesn't have write access to the uploads or logs folder.

    FIX

    $backendPath = "C:\JotFlow-Setup\2_APP\backend"
icacls "$backendPath\uploads" /grant "IIS_IUSRS:(OI)(CI)F" /T
icacls "$backendPath\logs"   /grant "IIS_IUSRS:(OI)(CI)F" /T
    ⛔ "permission denied for table users" (PostgreSQL)

    Cause: Database tables are owned by the postgres superuser instead of the jotflow app role. This happens after restoring a dump.

    FIX Run the ownership repair SQL (Admin PowerShell):

    $env:PGPASSWORD = "postgres"
psql -h localhost -U postgres -d jotflow -c "
GRANT ALL ON ALL TABLES IN SCHEMA public TO jotflow;
GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO jotflow;"
